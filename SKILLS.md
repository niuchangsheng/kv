# 技能要求 (Skills Required)

本文档列出了使用、开发和贡献本项目所需的各种技能。根据你的角色（用户、开发者、贡献者），你可能需要不同层次的技能。

## 用户技能

如果你只是想使用这个KV存储引擎，你需要以下基础技能：

### 必需技能

#### 1. C++ 基础
- **级别**: 初级到中级
- **要求**:
  - 理解C++基本语法（变量、函数、类）
  - 熟悉STL容器（string, vector, map等）
  - 理解指针和引用的使用
  - 了解基本的面向对象编程概念

#### 2. 编译和构建
- **级别**: 初级
- **要求**:
  - 能够使用CMake构建项目
  - 理解基本的编译错误信息
  - 能够在Linux/macOS/Windows上配置开发环境

#### 3. 版本控制基础
- **级别**: 初级
- **要求**:
  - 了解Git基本操作（clone, pull, checkout）
  - 能够从仓库获取代码

### 推荐技能

- **调试技能**: 能够使用GDB或IDE调试器
- **文档阅读**: 能够理解API文档和示例代码

## 开发者技能

如果你想深入理解项目实现或进行二次开发，你需要以下技能：

### 核心技能

#### 1. C++ 高级特性
- **级别**: 中级到高级
- **要求**:
  - **C++17标准**: 熟悉现代C++特性
    - 智能指针（unique_ptr, shared_ptr）
    - RAII原则
    - 移动语义（move semantics）
    - lambda表达式
    - auto关键字
  - **模板编程**: 基础模板使用
  - **STL深入**: 理解容器、迭代器、算法
  - **内存管理**: 理解堆栈内存、内存泄漏、智能指针

#### 2. 数据结构与算法
- **级别**: 中级
- **要求**:
  - 哈希表（Hash Table）原理和应用
  - 时间复杂度分析（O(1), O(log n), O(n)等）
  - 空间复杂度分析
  - 排序算法（快速排序、归并排序）
  - 二分查找

#### 3. 系统设计
- **级别**: 中级
- **要求**:
  - 理解KV存储系统的基本原理
  - 理解接口设计原则
  - 理解错误处理模式
  - 理解资源管理（RAII）

#### 4. 构建系统
- **级别**: 中级
- **要求**:
  - **CMake**: 
    - 理解CMakeLists.txt语法
    - 能够配置编译选项
    - 能够添加新的源文件
    - 理解target和依赖关系

#### 5. 测试框架
- **级别**: 中级
- **要求**:
  - **Google Test**:
    - 编写单元测试
    - 使用断言（ASSERT, EXPECT）
    - 测试组织和测试套件
    - Mock对象（如需要）

### 推荐技能

#### 1. 性能优化
- 理解性能瓶颈分析
- 了解内存对齐、缓存友好设计
- 理解编译器优化选项

#### 2. 并发编程
- 理解多线程基础概念
- 理解锁机制（读写锁、互斥锁）
- 了解无锁编程基础

#### 3. 调试和工具
- **调试器**: GDB熟练使用
- **性能分析**: Valgrind, perf等工具
- **内存检查**: AddressSanitizer, Valgrind

## 贡献者技能

如果你想为项目贡献代码，除了开发者技能外，还需要：

### 必需技能

#### 1. Git 高级操作
- **级别**: 中级
- **要求**:
  - 分支管理（branch, merge, rebase）
  - 提交规范（commit message规范）
  - Pull Request流程
  - 代码审查（Code Review）

#### 2. 代码质量
- **级别**: 中级
- **要求**:
  - **代码风格**: 遵循项目代码规范
  - **代码审查**: 能够审查他人代码
  - **文档编写**: 能够编写清晰的代码注释和文档

#### 3. 测试驱动开发
- **级别**: 中级
- **要求**:
  - 为新功能编写测试
  - 理解测试覆盖率
  - 能够修复测试失败

### 高级技能（可选）

#### 1. 存储系统深入
- **级别**: 高级
- **要求**:
  - LSM-Tree数据结构
  - WAL (Write-Ahead Logging)
  - SSTable文件格式
  - Compaction策略
  - 崩溃恢复机制

#### 2. 网络编程
- **级别**: 高级
- **要求**:
  - **gRPC**: 
    - Protocol Buffers定义
    - gRPC服务实现
    - 异步调用
  - 网络协议基础（TCP/IP）

#### 3. 分布式系统
- **级别**: 高级
- **要求**:
  - 一致性协议
  - 分布式事务
  - 副本管理

#### 4. 性能工程
- **级别**: 高级
- **要求**:
  - 性能基准测试
  - 性能调优
  - 内存分析
  - CPU分析

## 技能学习路径

### 路径1: 从用户到开发者

```
1. C++基础
   └─► 学习C++语法和STL
       └─► 实践：编写简单的KV操作程序

2. 构建系统
   └─► 学习CMake基础
       └─► 实践：修改CMakeLists.txt添加新文件

3. 数据结构
   └─► 学习哈希表、排序算法
       └─► 实践：实现简单的数据结构

4. 测试框架
   └─► 学习Google Test
       └─► 实践：为现有功能添加测试

5. 系统设计
   └─► 学习接口设计原则
       └─► 实践：设计新的功能接口
```

### 路径2: 从开发者到贡献者

```
1. Git高级操作
   └─► 学习分支管理和PR流程
       └─► 实践：提交第一个PR

2. 代码审查
   └─► 学习代码审查技巧
       └─► 实践：审查他人代码

3. 存储系统深入
   └─► 学习LSM-Tree、WAL等概念
       └─► 实践：实现持久化存储

4. 性能优化
   └─► 学习性能分析工具
       └─► 实践：优化现有代码性能
```

## 贡献指南

如果你具备相应技能并想贡献代码，请参考：

1. **阅读文档**: 先阅读 `README.md` 和 `docs/DESIGN.md`
2. **理解代码**: 熟悉现有代码结构和设计
3. **选择任务**: 从简单的bug修复或文档改进开始
4. **编写测试**: 确保新代码有测试覆盖
5. **提交PR**: 遵循项目的PR流程
